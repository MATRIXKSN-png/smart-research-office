أريد منك بناء المشروع على مراحل متدرجة وواضحة ومرتبة، مع الالتزام الصارم بالتدقيق في الكود ومنع أخطاء الاتصال أو الاستيراد أو التصدير.

اسم التطبيق:
المكتب الذكي للبحث العلمي

الفكرة الأساسية:
التطبيق يمثل مكتبًا ذكيًا للبحث العلمي يحتوي على عدة موظفين ذكاء اصطناعي، وكل موظف مسؤول عن مهمة محددة في معالجة الكتب والوثائق من لحظة رفع الملفات إلى لحظة عرض النتائج للمستخدم.

المطلوب الآن:
واجهة كاملة فقط باستخدام:
- React + Vite + TypeScript
- Tailwind CSS
- RTL كامل

ممنوع في هذه المرحلة:
- أي backend
- أي API
- أي proxy
- أي fetch
- أي axios
- أي localhost
- أي 127.0.0.1
- أي env vars خاصة بالاتصال

يجب أن يعتمد المشروع بالكامل على:
- useState
- mock data محلية فقط
- local components
- local helpers

المرحلة الأولى: الشكل العام والثيم والهوية البصرية

متطلبات الشكل:
- الواجهة عربية بالكامل

- أفضل من الصور من حيث المحاذاة والتناسق والمسافات
- الاتجاه RTL في التطبيق كله

المرحلة الثالثة: بناء الهيكل العام Layout
━━━━━━━━━━━━━━━━━━

أنشئ Layout موحد يحتوي على:
- Header علوي
- Sidebar على اليمين
- Main Content Area

الدعم المطلوب:
- تصميم responsive بالكامل
- يعمل على الكمبيوتر والهاتف معًا
- لا تختفي أزرار التنقل في الهاتف

سلوك الهاتف:
- في الشاشات الصغيرة يتحول Sidebar إلى Drawer أو قائمة قابلة للفتح والإغلاق
- يجب وجود زر واضح أعلى الصفحة لفتح القائمة في الهاتف
- يجب ألا تختفي روابط:
  - ربط الملفات بمرجع
  - النصوص المستخرجة
  - تحليل السؤال
  - الإعدادات

تبديل وضع العرض:
أضف زرًا واضحًا في أعلى الواجهة لتبديل وضع العرض يدويًا بين:
- وضع الكمبيوتر
- وضع الهاتف

سلوك زر التبديل:
- عند تفعيل وضع الهاتف يتم تطبيق تخطيط هاتف حتى لو كانت الشاشة كبيرة
- عند إيقافه يعود التخطيط الكامل الخاص بالكمبيوتر
- هذا التبديل بصري فقط داخل الواجهة
- استخدم state محلية فقط
- لا تستخدم أي backend أو إعدادات خارجية

محتوى الـ Header:
- عنوان التطبيق: المكتب الذكي للبحث العلمي
- وصف صغير: إدارة المراجع، استخراج النصوص، والتحليل الذكي في واجهة واحدة
- شارة حالة: النظام متصل
- شارة تاريخ عربية
- زر فتح القائمة للموبايل
- زر تبديل وضع العرض بين الهاتف والكمبيوتر

محتوى الـ Sidebar:
- بطاقة علوية باسم التطبيق ووصف مختصر
- روابط التنقل الرئيسية:
  1) ربط الملفات بمرجع
  2) النصوص المستخرجة
  3) تحليل السؤال
  4) الإعدادات

- الرابط النشط يكون واضحًا بتدرج بنفسجي

أسفل الروابط:
أضف قسمًا بعنوان:
حالة الوكلاء الذكية

بطاقات الوكلاء:
- مدير المكتب — نشط الآن
- موظف استخراج النصوص — نشط الآن
- موظف OCR — نشط الآن
- موظف التحليل — في وضع الانتظار
- موظف البحث — نشط الآن
- موظف التنظيم — نشط الآن

تعليمات:
- اجعل هذه البطاقات تبدو مثل موظفين داخل مكتب رقمي ذكي
- في الهاتف يجب ألا تختفي القائمة نهائيًا
- يجب أن تظهر من زر menu أعلى الصفحة أو كشريط تنقل مبسط واضح

المرحلة الرابعة: منطق الواجهة العام حسب فكرة البرنامج
━━━━━━━━━━━━━━━━━━

البرنامج يمثل مكتبًا ذكيًا للبحث العلمي.

الوكلاء:
1. مدير المكتب (Manager Agent)
2. موظف استخراج النصوص (Extraction Agent)
3. موظف OCR للصور (OCR Agent)
4. موظف تحليل النص (Analysis Agent)
5. موظف البحث في الإنترنت (Research Agent)
6. موظف تنظيم النتائج (Organization Agent)

يجب أن تعكس الواجهة بصريًا ووظيفيًا أن:
- مدير المكتب يستقبل الملفات
- يحدد نوعها
- يوزعها على الوكلاء
- تُعرض النتائج النهائية للمستخدم

مهم:
- نفذ كل شيء ببيانات mock محلية فقط
- لا API
- لا backend
- لا اتصال شبكي

المرحلة الخامسة: صفحة ربط الملفات بمرجع
━━━━━━━━━━━━━━━━━━

هذه الصفحة تمثل نقطة دخول الملفات إلى المكتب الذكي.

المطلوب:
- عنوان رئيسي واضح
- وصف يشرح أن النظام يربط الملفات بمرجع واحد قبل توزيع المهام على الوكلاء

أنشئ بطاقة رئيسية منقسمة إلى قسمين.

القسم الأول:
- عنوان فرعي: رفع الملفات وربطها بالمرجع
- وصف صغير
- حقل اسم الكتاب
- حقل اسم المؤلف
- منطقة UploadDropzone أنيقة
- نص مساعد يوضح أن الملفات يمكن أن تكون PDF أو صور JPG/PNG
- زر اختيار الملفات
- زر أساسي: ربط الملفات
- زر ثانوي: إعادة تعيين

القسم الثاني:
- بطاقة إحصاءات تحتوي على:
  - عدد الملفات
  - الصور المرسلة
  - نوع الرفع

- بطاقة حالة تعرض نصًا مثل:
بعد رفع الملفات سيظهر هنا ملخص حالة معالجة المرجع وتوزيع الملفات على الوكلاء

أسفل البطاقة الرئيسية:
- قسم بعنوان: الملفات المختارة
- اعرض EmptyState مناسب إذا لم توجد ملفات
- أضف mock files عربية

أضف أيضًا:
مسارًا بصريًا صغيرًا يوضح انتقال الملف بين الوكلاء:
Manager → Extraction/OCR → Analysis → Research → Organization

━━━━━━━━━━━━━━━━━━
المرحلة السادسة: صفحة النصوص المستخرجة
━━━━━━━━━━━━━━━━━━

هذه الصفحة مخصصة لعرض النصوص الأصلية المستخرجة بدون أي تعديل.

المطلوب:
- عنوان الصفحة
- وصف يؤكد أن النص الأصلي محفوظ كما ورد في المصدر
- شارة أو تنبيه بصري واضح يقول:
يُعرض النص كما هو دون إعادة صياغة

أضف زرًا بارزًا أعلى الصفحة:
حفظ جميع النصوص لهذا المرجع

قسّم الصفحة إلى عمودين.

العمود الأول الأضيق:
- بطاقة بعنوان: قائمة المراجع
- اعرض مراجع mock محلية

كل مرجع يجب أن يحتوي على:
- اسم المرجع
- معلومات مختصرة عنه إن وجدت
- زر حذف واضح أمامه أو بمحاذاته

زر الحذف:
- استخدم أيقونة Trash مناسبة
- يكون صغيرًا وأنيقًا وواضحًا
- عند hover يظهر بشكل أوضح لكن بدون إزعاج بصري
- لونه تحذيري متناسق مع الثيم البنفسجي
- مثال:
  - خلفية وردية أو بنفسجية فاتحة جدًا
  - أيقونة أو نص بلون أحمر هادئ أو خمري

عند الضغط على زر الحذف:
- افتح نافذة ConfirmDialog بسيطة باللغة العربية

نص التأكيد:
هل تريد حذف هذا المرجع من القائمة؟

أزرار النافذة:
- إلغاء
- حذف المرجع

عند تأكيد الحذف:
- يتم حذف المرجع محليًا باستخدام state فقط
- يتم حذف الصفحات أو العناصر التابعة له من العرض الحالي أيضًا

إذا كان المرجع المحذوف هو المرجع المحدد حاليًا:
- اختر المرجع التالي تلقائيًا إن وجد
- أو اعرض EmptyState مناسب إذا لم تبق مراجع

إذا تم حذف آخر مرجع:
اعرض EmptyState بالنص:
- لا توجد مراجع حاليًا
- يمكنك إضافة ملفات جديدة لبدء العمل

مهم:
- تأكد من عدم حدوث أي runtime error عند حذف آخر مرجع
- نظف الحالة المرتبطة بالمرجع المحذوف

عند اختيار مرجع من القائمة:
- تظهر تحته قائمة الصفحات أو الملفات التابعة له
- العنصر المحدد يكون واضحًا

العمود الثاني الأوسع:
- شريط إجراءات يحتوي على:
  - نسخ النص
  - حفظ كملف وورد

- بطاقة معلومات صغيرة تعرض:
  - اسم الملف
  - اسم المرجع
  - الصفحة
  - طريقة الاستخراج
  - اسم الوكيل المنفذ
  - الحالة

- مساحة كبيرة لعرض النص الأصلي كما هو

أضف فصلًا بصريًا واضحًا بين:
- النص الأصلي
- قسم آخر أسفله بعنوان:
تحليل منفصل عن النص الأصلي

إذا لم يوجد نص:
اعرض EmptyState بالنص:
لا يوجد نص مستخرج لهذه الصفحة

تعليمات:
- اجعل تصميم الصفحة يوحي أن النص الأصلي غير قابل للتحرير
- النص محفوظ كما ورد في المصدر

المرحلة السابعة: صفحة تحليل السؤال
━━━━━━━━━━━━━━━━━━

هذه الصفحة تمثل دور الذكاء الاصطناعي في فهم السؤال والبحث داخل النصوص الأصلية.

أضف:
- عنوانًا رئيسيًا
- وصفًا:
اكتب سؤالك البحثي ليقوم المكتب الذكي بالبحث داخل النصوص الأصلية وعرض الفقرات الأكثر صلة

قسّم الصفحة إلى عمودين.

العمود الأيمن الأضيق:
- textarea بعنوان: سؤالك البحثي
- select بعنوان: نطاق البحث

خيارات النطاق:
- جميع المراجع
- مرجع محدد
- صفحات محددة

- بطاقة بعنوان: تحليل ذكي (LLM)
- بداخلها ToggleSwitch لتفعيل أو تعطيل التحليل الذكي

- بطاقة بعنوان: إعدادات المطابقة
تعرض عناصر مثل:
- عدد النتائج الافتراضي: 5
- الوضع الصارم للنص الأصلي: مفعل
- OCR: مفعل
- التعقيب الخاص بكل فقرة: مفعل
- التعقيب الكلي: مفعل
- الاستنتاجات النهائية: مفعل

- زر أساسي كبير:
ابدأ تحليل السؤال

العمود الأيسر الأوسع:
- مساحة النتائج

في الحالة الافتراضية:
- اعرض EmptyState
- عنوان: مساحة النتائج
- وصف مناسب

عند وجود نتائج mock:
- اعرض السؤال في الأعلى
- اعرض بطاقات نتائج مرتبة حسب الصلة

كل بطاقة يجب أن تحتوي على:
- عنوان فرعي أو موضوع
- الفقرة الأصلية كما هي بدون أي تعديل
- المرجع
- الصفحة
- درجة الصلة بالسؤال
- تحليل منفصل قصير أسفل الفقرة

مهم جدًا:
- يجب أن يكون واضحًا بصريًا أن التحليل منفصل عن النص الأصلي
- أضف ملاحظة بصرية ثابتة تقول:
لا يتم تعديل النص الأصلي، ويعرض كما ورد في المصدر

المرحلة الثامنة: صفحة الإعدادات
━━━━━━━━━━━━━━━━━━

يجب أن تحتوي الصفحة على:
- عنوان رئيسي
- وصف صغير

إعدادات الاستخراج والبحث
ويحتوي على:
- Toggle لتفعيل تقنية OCR

القسم الثاني:
إعدادات الذكاء الاصطناعي
ويحتوي على:
- Toggle لتفعيل ChatGPT أو LLM
- وصف صغير يوضح أن دوره تحليلي فقط وليس إعادة كتابة النص

القسم الثالث:
إعدادات عرض التعقيبات والاستنتاجات
ويحتوي على:
- Toggle لتفعيل التعقيب بين الفقرات المتشابهة
- Toggle لتفعيل التعقيب الكلي عن الفقرات المتشابهة
- Toggle لتفعيل الاستنتاجات في آخر الفصل
- Toggle لإظهار المرجع ورقم الصفحة دائمًا

القسم الرابع:
الثيمات والمظهر
- بطاقة placeholder منظمة
- تعرض معلومات بسيطة عن الثيم البنفسجي الحالي

أسفل الصفحة:
- زر أساسي كبير بعنوان:
حفظ الإعدادات

- صفحة الإعدادات

القواعد:
1) يجب الحفاظ على النص الأصلي كما هو
2) يمنع تعديل النص أو إعادة صياغته أو اختصاره أو تغيير معناه
3) التحليل يجب أن يكون منفصلًا بصريًا عن النص الأصلي
4) عند عرض النتائج المرتبطة بسؤال، يجب أن يظهر:
- السؤال
- الفقرات الأصلية
- المرجع
- الصفحة
- التحليل المنفصل
5) يجب أن تعكس الواجهة أن دور الذكاء الاصطناعي هو:
- فهم السؤال
- البحث داخل النصوص
- تحديد الفقرات المناسبة
- تقديم تحليل منفصل فقط
وليس:
- إعادة كتابة النص الأصلي
6) عند عرض الفقرات المرتبطة بالسؤال، اجعل ترتيبها حسب درجة الصلة

المرحلة الحادية عشرة: دعم الكمبيوتر والهاتف معًا
━━━━━━━━━━━━━━━━━━

يجب أن يعمل التطبيق بشكل ممتاز على:
- الكمبيوتر
- الهاتف

ممنوع:
- اختفاء أزرار التنقل
- اختفاء شريط المهام
- اختفاء صفحات الإعدادات أو تحليل السؤال أو غيرها في الهاتف

السلوك المطلوب:
- عند الانتقال إلى الهاتف تتحول القائمة الجانبية إلى Drawer أو قائمة قابلة للفتح بزر أعلى الصفحة
- أضف زرًا واضحًا في أعلى الواجهة لتبديل وضع العرض يدويًا بين:
  - وضع الكمبيوتر
  - وضع الهاتف

قواعد التبديل:
- عند تفعيل وضع الهاتف يجب أن تتصرف الواجهة كلها كتخطيط هاتف حتى على شاشة الكمبيوتر
- عند إيقافه تعود إلى تخطيط الكمبيوتر
- هذا التبديل يجب أن يكون محليًا باستخدام state فقط
- أضف أيضًا معالجة Responsive تلقائية حسب حجم الشاشة
- أعطِ أولوية لزر التبديل اليدوي إن كان مفعلًا

المرحلة الثانية عشرة: حقوق التصميم والبرمجة
━━━━━━━━━━━━━━━━━━

أضف في أسفل الواجهة Footer بسيطًا وأنيقًا، أو في أسفل الشريط الجانبي، نص حقوق واضح باللغة العربية:

حقوق التصميم والبرمجة: matrixksn

تعليمات:
- اجعل النص صغيرًا وراقيًا ومتناسقًا مع الثيم البنفسجي
- لا يزعج التصميم العام
- يمكن أيضًا إظهار الحقوق في صفحة الإعدادات ضمن قسم صغير بعنوان:
معلومات التطبيق

━━━━━━━━━━━━━━━━━━
المرحلة الثالثة عشرة: التدقيق ومنع الأخطاء
━━━━━━━━━━━━━━━━━━

هذا الجزء إلزامي جدًا.

ممنوع نهائيًا:
- fetch
- axios
- API calls
- proxy
- localhost
- 127.0.0.1
- env vars خاصة بالاتصال

استخدم فقط:
- useState
- mock data
- local helpers
- local components

دقق الكود قبل إنهائه:
- تأكد أن كل import صحيح
- تأكد أن كل export صحيح
- لا يوجد hook غير موجود
- لا يوجد component مستورد ولم يتم إنشاؤه
- لا يوجد مسار ملف خاطئ
- لا يوجد تعارض أسماء
- لا يوجد خطأ TypeScript
- لا يوجد خطأ Vite
- لا يوجد خطأ runtime
- لا يوجد خطأ module export
- لا يوجد ECONNREFUSED

تأكد أيضًا:
- أن وضع الهاتف يعمل دون كسر التخطيط
- أن وضع الكمبيوتر يعمل دون كسر التخطيط
- أن القائمة لا تختفي في الهاتف
- أن حذف المراجع لا يسبب أي خطأ
- أن المشروع يبنى ويعمل محليًا بدون أخطاء
