```text
أريد الآن تنفيذ تحديث شامل للمشروع بحيث يعمل فعليًا من لحظة رفع الملفات إلى استخراج النصوص وعرض النتائج، مع إضافة التكامل الذكي عبر الإنترنت عند توفر الشبكة، وإضافة زر الوضع الداكن في أعلى الواجهة.
نفّذ المطلوب على مراحل واضحة، مع أولوية قصوى للتشغيل الصحيح والاستقرار ومنع الأخطاء.
━━━━━━━━━━━━━━━━━━
أولًا: إصلاح منطق معالجة الملفات بالكامل
━━━━━━━━━━━━━━━━━━
المشكلة الحالية:
البرنامج يرفع الملفات ويربطها بالمرجع، لكن لا تتم معالجة الملفات بعد الرفع، ولا يتم استخراج أي نص، وتبقى الصفحات في حالة "في الانتظار".
أريد إصلاح هذا الخلل جذريًا.
المطلوب:
1) بعد الضغط على "ربط الملفات" وإنشاء المرجع، يجب أن تبدأ المعالجة تلقائيًا مباشرة بدون الحاجة إلى أي زر إضافي.
2) أنشئ pipeline معالجة واضح وفعلي كالتالي:
- Manager Agent يستقبل المرجع والملفات
- يحدد نوع كل ملف
- إذا كان PDF نصي → Extraction Agent
- إذا كان صورة أو PDF مصور → OCR Agent
- بعد انتهاء الاستخراج → Organization Agent يرتب النتائج
3) عرّف حالات ثابتة لكل ملف أو صفحة:
- queued = في الانتظار
- processing = قيد المعالجة
- extracted = تم الاستخراج
- failed = فشل الاستخراج
4) عند بدء المعالجة:
- تتحول الملفات أولًا إلى queued
- ثم processing
- ثم extracted أو failed
ويجب أن يظهر هذا التغير في الواجهة فعليًا.
5) أنشئ خدمة واضحة مثل:
- services/extraction/processingPipeline.ts
أو
- services/extraction/processReference.ts
وتكون مسؤولة عن:
- إنشاء page records
- تشغيل الاستخراج
- تحديث الحالة الموحدة
- حفظ النتائج
6) إذا لم يكن OCR الحقيقي أو backend النهائي جاهزًا بعد:
لا أريد أن يبقى التطبيق معلّقًا.
أنشئ fallback extraction محلي مؤقت يعمل على ملفات المستخدم الحالية فقط.
مثلًا:
- لكل صورة PNG/JPG أنشئ نصًا مستخرجًا تجريبيًا مرتبطًا باسم الملف واسم المرجع
- لكل PDF أنشئ نصًا أوليًا مرتبطًا بالملف نفسه
لكن ممنوع استخدام نصوص من كتب افتراضية لا تخص المستخدم.
7) المطلوب الآن ليس الكمال في OCR، بل أن تكتمل دورة العمل فعليًا:
- رفع الملف
- بدء المعالجة
- تغير الحالة
- ظهور نص مستخرج أو رسالة فشل
8) في صفحة النصوص المستخرجة:
- إذا كانت الصفحة queued يظهر:
  "هذه الصفحة في انتظار المعالجة."
- إذا كانت processing يظهر:
  "جاري استخراج النص من هذه الصفحة..."
- إذا كانت extracted يظهر النص الأصلي المستخرج
- إذا كانت failed يظهر:
  "تعذر استخراج النص من هذا الملف في الوقت الحالي."
9) عند اكتمال أول صفحة بنجاح:
- يتم تحديدها تلقائيًا
- ويظهر نصها مباشرة في مساحة العرض
10) أضف زرًا صغيرًا بعنوان:
- إعادة المعالجة
ليعيد تشغيل pipeline للمرجع أو للصفحة المختارة
11) كل صفحة مستخرجة يجب أن ترتبط بهذه البيانات:
- id
- referenceId
- fileId
- fileName
- pageNumber
- extractionMethod
- agentName
- status
- originalText
- analysisText
12) إذا رفع المستخدم عدة صور:
- كل صورة تتحول إلى صفحة مستقلة
- لكل صفحة حالة مستقلة
- لكل صفحة نص مستقل
13) احفظ نتائج المعالجة في localStorage
حتى لا تعود الحالات إلى "في الانتظار" بعد تحديث الصفحة
14) تأكد أن الرسالة:
"لا يوجد نص مستخرج لهذا المرجع بعد"
لا تظهر إذا كانت هناك صفحات extracted فعلًا
15) افحص المشروع وتأكد أن الواجهة تقرأ من نفس المصدر الذي تكتب فيه المعالجة
ولا يوجد تضارب بين:
- files
- pages
- extractedPages
- text
- originalText
- content
━━━━━━━━━━━━━━━━━━
ثانيًا: ربط اختياري بخدمات الذكاء الاصطناعي عند توفر الشبكة
━━━━━━━━━━━━━━━━━━
أريد أن يعمل التطبيق في وضعين:
1) وضع محلي Local Mode
2) وضع متصل AI Online Mode
الفكرة:
إذا توفرت الشبكة وتم تفعيل الخيار من الإعدادات، يستفيد التطبيق من مواقع أو خدمات الذكاء الاصطناعي لزيادة دقة:
- تحليل السؤال
- استخراج الكلمات المفتاحية
- تنظيم النتائج
- كتابة التعقيب المنفصل
- تحسين الاستنتاجات
لكن مع شرط صارم:
النص الأصلي المستخرج لا يتم تعديله أبدًا، ولا إعادة صياغته، ولا اختصاره.
المطلوب:
1) أنشئ طبقة تكامل واضحة ومنفصلة مثل:
- services/ai/
وتكون قابلة للتوسعة
2) ادعم بنية مزودات مثل:
- OpenAI
- OpenRouter
- Gemini
- Claude
ويكفي البدء بمزود واحد فعليًا لكن البنية تكون جاهزة للتوسعة
3) أضف إعدادات مركزية تحتوي على:
- aiEnabled
- aiProvider
- apiKeyConfigured
- networkAvailable
- fallbackToLocal
4) إذا لم توجد شبكة أو لم يوجد API Key أو فشل الاتصال:
- لا ينهار التطبيق
- يعود تلقائيًا إلى Local Mode
- تظهر رسالة عربية واضحة:
  "تعذر الوصول إلى خدمة الذكاء الاصطناعي، تم التحويل إلى المعالجة المحلية."
5) لا تضع مفاتيح API مباشرة في الواجهة الأمامية بشكل غير آمن.
إن أمكن اجعل التكامل عبر route آمن أو adapter layer منظمة.
6) في صفحة تحليل السؤال:
أضف شارة توضح:
- تم التحليل محليًا
أو
- تم تحسين التحليل عبر خدمة ذكاء اصطناعي
7) في حالة الوكلاء:
- موظف التحليل: متصل بالخدمة الذكية / وضع محلي
- موظف البحث: متصل بالويب / غير متصل
8) داخل صفحة الإعدادات أضف قسمًا بعنوان:
إعدادات التكامل الذكي
ويحتوي على:
- Toggle لتفعيل الذكاء الاصطناعي المتصل
- Select لاختيار المزود
- حقل API Key
- Toggle للعودة التلقائية إلى الوضع المحلي عند الفشل
- زر اختبار اتصال
- شارة حالة الاتصال
9) عند الضغط على "اختبار اتصال":
- إذا نجح الاتصال تظهر رسالة:
  "تم الاتصال بنجاح بخدمة الذكاء الاصطناعي"
- إذا فشل:
  "فشل الاتصال، سيتم استخدام الوضع المحلي"
10) أضف prompts داخلية واضحة تمنع الخدمة الذكية من:
- إعادة كتابة النص الأصلي
- اختصار النص الأصلي
- تغيير معنى النص الأصلي
ويقتصر دورها على:
- التحليل المنفصل
- التعقيب
- الاستنتاج
- اقتراح المراجع
11) عالج الحالات التالية:
- timeout
- network failure
- invalid key
- provider unavailable
- rate limit
وفي جميعها:
- لا ينهار التطبيق
- يعود للوضع المحلي إذا كان fallback مفعّلًا
━━━━━━━━━━━━━━━━━━
ثالثًا: إضافة زر الوضع الداكن بالأعلى
━━━━━━━━━━━━━━━━━━
أريد إضافة زر واضح في أعلى الواجهة داخل الـ Header للتحكم بالمظهر.
المطلوب:
1) أضف زرًا في الأعلى بعنوان مناسب مثل:
- الوضع الداكن
أو
- المظهر
2) عند الضغط عليه:
- يبدل بين Light Mode و Dark Mode مباشرة
3) يجب أن يؤثر على كامل التطبيق:
- الخلفية
- البطاقات
- القائمة الجانبية
- النصوص
- الأزرار
- الحقول
- النوافذ المنبثقة
- حالات hover و active
4) حافظ على الهوية البنفسجية نفسها لكن بنسختين:
ألوان فاتحة:
- Background: #F6F3FF
- Surface: #FFFFFF
- Surface Soft: #FCFAFF
- Sidebar Start: #2E1065
- Sidebar End: #4C1D95
- Primary: #6D28D9
- Text Main: #231942
- Border: #E9D5FF
ألوان داكنة:
- App Background: #0F0B1A
- Surface: #171228
- Surface Soft: #1E1830
- Sidebar Start: #140A2E
- Sidebar End: #2A115A
- Primary: #8B5CF6
- Text Main: #F5F3FF
- Border: #31254F
5) لا أريد مجرد عكس ألوان، بل ثيم احترافي:
- تباين ممتاز
- وضوح عالٍ
- راحة بصرية
- استمرارية في الهوية البنفسجية
6) احفظ اختيار الوضع محليًا حتى يبقى بعد تحديث الصفحة
7) تأكد أن الهاتف والكمبيوتر كلاهما يدعمان الوضع الداكن دون كسر التخطيط
━━━━━━━━━━━━━━━━━━
رابعًا: شروط بنيوية مهمة جدًا
━━━━━━━━━━━━━━━━━━
1) افصلوا المنطق إلى طبقات واضحة:
- services/extraction
- services/analysis
- services/ai
- services/storage
- contexts
- pages
- components
2) لا تخلطوا منطق المعالجة أو الذكاء الاصطناعي داخل مكونات العرض مباشرة
3) تأكدوا من:
- عدم وجود أخطاء import/export
- عدم وجود hooks مفقودة
- عدم وجود module not found
- عدم وجود TypeScript errors
- عدم وجود runtime errors
4) إذا لم توجد شبكة أو API Key:
- يعمل التطبيق محليًا بشكل طبيعي
5) إذا لم يكن OCR الحقيقي جاهزًا:
- استخدموا fallback محلي منظم بدل بقاء الصفحات معلّقة
6) عند حذف مرجع:
- تحذف صفحاته ونصوصه وحالاته التابعة
- لا تظهر بيانات وهمية قديمة
━━━━━━━━━━━━━━━━━━
خامسًا: السيناريو النهائي المطلوب
━━━━━━━━━━━━━━━━━━
أريد أن أستطيع اختبار هذا التسلسل:
1) أكتب اسم الكتاب والمؤلف
2) أرفع عدة ملفات أو صور
3) أضغط ربط الملفات
4) ينشأ المرجع
5) تبدأ المعالجة تلقائيًا
6) تتحول الحالات من:
في الانتظار → قيد المعالجة → تم الاستخراج أو فشل الاستخراج
7) عند نجاح صفحة يظهر النص مباشرة
8) أذهب لتحليل السؤال
9) إذا كانت الشبكة متوفرة والتكامل مفعّل يتم تحسين التحليل عبر خدمة AI
10) إذا فشل الاتصال يعود للوضع المحلي تلقائيًا
11) أبدل الوضع الداكن من الزر العلوي
12) تبقى الإعدادات والبيانات محفوظة بعد إعادة تحميل الصفحة
━━━━━━━━━━━━━━━━━━
سادسًا: أولوية التنفيذ
━━━━━━━━━━━━━━━━━━
نفّذوا العمل بهذا الترتيب:
1) إصلاح pipeline معالجة الملفات
2) عرض النصوص المستخرجة وحالاتها بشكل صحيح
3) حفظ النتائج محليًا
4) إضافة التكامل الاختياري مع خدمات الذكاء الاصطناعي
5) إضافة زر الوضع الداكن بالأعلى
6) تدقيق كامل على المشروع
```
